Bias – Variance Trade Off

Section i

Bias :-

• Bias is the error caused by overly simple assumptions in the model.
A high-bias model:
a. Is too simple
b. Misses important patterns
• Makes consistent mistakes
Example Trying to fit a straight line to data that is actually curved.
Bias = "How wrong are we on average?"
Bias comes from oversimplifying.
The model assumes the world is simpler than it really is and consistently misses the mark.
Bias = Underfitting
Section ii

Variance :-

– Variance is the error caused by the model being too sensitive to training data.
A high-variance model:
a. Learns noise instead of correct pattern
b. Changes a lot with different datasets
c. Performs well on training data but poorly on unseen data
Model is "too jumpy"
– The model memorises noise & behaves wildly on new data (overfitting)
– Variance causes overfitting problems.
Variance = "How much does the model change with different unseen data?"
Section iii

Underfitting :-

– Underfitting happens when the model is too simple to learn data.
– Poor performance on both training and test data.
e.g. Using a straight line for highly non-linear data.
Section iv

Overfitting :-

– Overfitting happens when the model learns too much from training data (including noise)
– When low bias & High Variance
– Here very accurate on training data, poor performance on test data
– Memorizing answers instead of understanding concepts.
Core Concept

# Bias – Variance Trade Off :-

Balance between too simple (high bias) and too complex (high variance) — minimize the total error, not just one of them.
Total Error = Bias² + Variance + Irreducible Noise
– If reduce bias (more complex model) → variance tends to increase, and vice-versa.
– Goal: find the sweet middle that minimises total error.
Notes –
1. Linear Regression
Usually low variance; higher bias. Works when the underlying pattern is simple.
2. Deep Neural Network
Low bias, can have high variance. Needs lots of data + Regularization.
3. Random Forest
Averages many trees to reduce variance. Often a great generalizer.
Tips

Practice Tips :-

  1. Use Regularization (L1/L2), dropout to reduce variance.
  2. Get more data to help complex model generalize better.
  3. Use Cross-Validation to estimate generalization error reliably.
  4. Ensemble methods (Bagging / Boosting) often reduce variance.
SMOTE

Ideologies / SMOTE :-

• SMOTE stands for "Synthetic Minority Over-Sampling Technique".
• Used for handling imbalanced datasets — when one class (usually important one) has far fewer examples than others.
• Model trained on such data tend to ignore minority class and become biased toward majority class.
• It generates new data points by interpolating between existing datasets.

Ideologies for Imbalanced Data:

  1. If dataset is small → Use oversampling (SMOTE)
  2. If dataset is too large → Use undersampling or balanced RF
  3. If model is tree-based → Use class weights or balanced ensembles
  4. If imbalance is extreme (1%) → Try anomaly detection
