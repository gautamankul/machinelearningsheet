Feature Scaling

# StandardScaler :-

•StandardScaler ensures all features are on the same scale (mean = 0, std = 1).
Why Scale?
•Why? — To make sure no features dominate just because of its size.
FeatureValues
Age20, 30, 40
Salary20,000 · 80,000 · 1,50,000
⚠ Problem Salary is much larger than Age → it may cause following problems:
1. Model Becomes Biased
ŷ = w₁·age + w₂·salary

Since w₂ dominates, Age may be ignored.
2. Gradient Descent Unstable
Cost function becomes skewed (elongated shape). Gradient descent:
• Zig-zags
• Takes longer to converge
• Might even fail
Formula
•Normalisation of trained data / feature are required for: Linear Regression · Logistic · SVM · KNN · PCA
Z-Score Standardisation Formula z = (x − μ) / σ

where   μ = mean  ·  σ = standard deviation
How We Use

Scaler Methods :-

1 fit()
Learn parameters (does NOT change data).
Internally calculates mean (μ) & standard deviation (σ).

scaler.fit(X_train)
→ shows nothing when printed
2 transform()
Apply transformation using learned values.
Computes features using previously learned mean & std.

scaler.transform(X)
→ [[-1., -1.], [1., 1.]]
3 fit_transform()
Do both together — learn + apply in one step. Use on training data only.

Example Usage:

from sklearn.preprocessing import StandardScaler scaler = StandardScaler() # eg: X = [[2000, 6000], [8000, 12000]] # fit — learns mean & std (shows nothing) y = scaler.fit(X) print(y) # shows nothing # transform — applies scaling print(scaler.transform(X)) # [[-1., -1.], [1., 1.]] # Standard usage scaled_X_train = scaler.fit_transform(X_train) # training scaled_X_test = scaler.transform(X_test) # testing
Critical Notes
⚠ Training vs Testing Scaling Rule • If you are training scaling → must use fit_transform() method.
• But in testing scaling → must use transform() method only,
because it makes the model learn from test data — which it is NOT supposed to see.
Advanced

Does LR have Hyperparameter Tuning?

–Basic Ordinary Least Regression has almost no hyperparameters.
–But once we use Regularization, optimization methods, or feature transformations → tuning becomes very important.
Main Hyperparameters in Linear Regression:
a. Regularization Strength
Controls model complexity.
RidgeLassoElastic Net
b. Learning Rate (α)
Used in Gradient Descent. Controls step size during optimization.
c. No. of Iterations
How many times model trains (epochs).
d. Polynomial Degree
Controls model flexibility — higher degree = more complex curve.
e. Feature Selection Choices
Which features to include/exclude from model.
Tuning Methods
Linear Regression Hyperparameter Tuning Methods:
  1. a Grid Search — exhaustive search over param grid
  2. b Random Search — random sampling over param space
  3. c Cross-Validation (cross_val_score) — reliable error estimate
  4. d ★ Regularization Path Method
Full Example

# GridSearchCV Pipeline :-

from sklearn.model_selection import train_test_split, GridSearchCV from sklearn.preprocessing import StandardScaler from sklearn.linear_model import Ridge from sklearn.pipeline import Pipeline from sklearn.metrics import r2_score # Train-test split X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.2) # Build Pipeline pipeline = Pipeline([ ('scaler', StandardScaler()), ('model', Ridge()) ]) # Define Hyperparameters params = {'model__alpha': [0.01, 0.1, 1, 10, 100]} # Apply Grid Search + Cross Validation grid = GridSearchCV(pipeline, params, cv=5, scoring='r2') grid.fit(X_train, y_train) # Best param print('Best alpha :', grid.best_params_) # Evaluate on Test Data best_model = grid.best_estimator_ y_pred = best_model.predict(X_test) print("R² Score", r2_score(y_test, y_pred))
