≑Multi-Linear Regression

In Simple regression, we predict a value using one input (feature):

Y = mx + b

But, in Multiple linear regression, we use multiple inputs:

y = β₀ + β₁x₁ + β₂x₂ + · · · + βₙxₙ
β₀= Intercepter (bias)
β₁, β₂= Coefficient / slope / variance
x₁, x₂= features value
Intuition:—

Instead of predicting something with just one factor, you use many factors together.

Example — Predicting House Price:

  • Size (x₁)
  • No. of Bedrooms (x₂)
  • Location Score (x₃)
Note:—When you use or add two or more variables, it may always increase the R² evaluation metrics.
#Geometric Understanding
  • In Simple Regression → a line
  • Multiple Regression → a plane (or hyperplane in higher dimensions)
  • Still coefficients obtained by minimizing the sum of square error (MSE)
#New Considerations
  • Model may 'overfit' or become complex.
  • High train accuracy but low test accuracy.
  • Multi-collinearity can reduce accuracy.
#How it Works
  • MLR finds the best coefficients (β values) that minimize error.
  • Minimize Mean Square Error:
MSE = 1n Σ (yactual − ypredicted)²
The model adjusts β₁, β₂… until predictions are as close as possible.
#Matrix Form

Instead of writing long equations, we use matrix form:

Y = XB + ε
Y= output vector
X= input matrix
β= coefficients
ε= Error
Solution (Normal Equation):
β = (XTX)-1 × XTY

This is the Mathematical backbone of MLR.

#Assumptions of MLR
  1. 1.Linearity
  2. 2.No Multi-collinearity
  3. 3.Homoscedasticity
1.Multi-Collinearity
  • When input variables are highly correlated, that phenomena is known as multi-collinearity.
  • Model gets confused, coefficients become unstable.
Before applying the Multi-collinearity fixing technique, we can check a particular feature and how much is required for our model. That concept is p-value.
#p-value
  • In Regression, the p-value tells us whether a feature has a significant relationship with the dependent variable.
  • The p-value tests null hypothesis that the coefficient of a predictor is equal to zero (no effect).
Assumptions:—
  • Low p-value (< 0.5) → The feature significantly affects.
  • High p-value (> 0.5) → The feature does not have significant affects.
FeatureCoefficientp-value
Size50000.09
Bedrooms20000.04
Distance to park1000.6
Formula:—

It comes from a t-test:

t = β̂ − 0SE(β̂)
β̂= estimated coefficient
SE= Standard Error
Note:Then this t-statistic is converted into a p-value.

eg:—

import statsmodel.api as sm
X = data[['size', 'Bedrooms']]
y = data['price']
X = sm.add_constant(X) # add Intercept (β₀)
model = sm.OLS(y, X).fit()
print(model.summary()) or print(model.pvalues)

Output: Coef, Std Err, t, P>|t| columns shown for const, Size, Bedrooms

#How to Detect Multi-Collinearity
  1. a.Variance Inflation Factor (VIF)
  2. b.Correlation Matrix / Heatmap
  3. c.Condition Number
  4. d.Unstable Coefficient Signs
1. Variance Inflation Factor (VIF):—
  • VIF measures how much the variance of a coefficient is inflated due to multi-collinearity.
VIFj = 11 − R²j
R²jR² from regressing feature Xj on all other features.
Intuition:—
  • If VIF > 5 (sometimes > 10) → indicates High Multi-collinearity.
  • If xj is independent → R²j ≈ 0 → VIF ≈ 1.
  • So VIF directly quantifies redundancy in features space.

eg:—

from statsmodels.stats.outliers_influence import variance_inflation_factor
X = df.drop(columns=['salary'])
y = df['price']
X = sm.add_constant(X)
vif = pd.DataFrame()
vif['VIF'] = [variance_inflation_factor(X.values, i)
              for i in range(X.shape[1])]
vif['variable'] = X.columns
print(vif)
Important: Interpretation: Both Years Experience & Age have ≈ 39.5 which is much higher — meaning they are highly correlated to each other.
2.Correlation Matrix / Heat Map
  • Correlation matrix only captures pairwise relationships.
  • And it solves hidden multi-collinearity problems: low pairwise correlations but still strong collinearity.
Pairwise Correlation:—
ρᵢⱼ = Cov(xᵢ · xⱼ)σᵢ σⱼ
When to use:—
  1. 1.Quick Screening
  2. 2.Feature pruning before modeling
Note:—Not sufficient alone for serious modeling.

eg:—

df_corr = df.corr()
threshold = 0.2
high_corr = []

for col1 in df_corr.columns:
  for col2 in df_corr.columns:
    if col1 != col2 and abs(df_corr.loc[col1, col2]) > threshold:
      high_corr.append((col1, col2, df_corr.loc[col1, col2]))
3.Using Condition Number

Condition number of matrix X:

κ(X) = σmaxσmin
σmaxLargest singular values
σminSmallest singular values

Also known as Numerical Stability View.

Intuition:—
  1. 1.Measures sensitivity of solution to small changes.
  2. 2.If smallest singular values ≈ 0 → matrix nearly singular.
Note:—Small noise in data → huge changes in coefficient.
4.Using Eigenvalues / Eigen Decomposition
import numpy as np

Q.How do you Handle Multi-Collinearity?
  1. 1.Check correlation + VIF
  2. 2.If small problem → remove variables
  3. 3.If large / high-dimensional:
        Use Ridge: use for coefficient stability.
  4. 4.If feature selection needed → Use Lasso
  5. 5.When dealing with high-dimensional correlated data → Use PCA
#Multi-Collinearity Solutions
  1. 1.Remove Variables
  2. 2.Use PCA (Principal Component Analysis)
  3. 3.Use Regularization (Ridge / Lasso)
2.PCA — Principal Component Analysis
  • It is an unsupervised learning but here we will see the point of multi-collinearity.
  • PCA is a technique to reduce the no. of features (dimensions) while keeping as much important information as possible.

eg:— Think of a dataset like this:

  • Height
  • Weight
  • Age
  • Income
  • Spending Scale

Some of these are correlated (eg: height & weight)

PCA finds:

  • The most important directions in the data
  • Remove redundancy
  • Keeps only what matters
#How PCA Works
  1. 1.First Principal Component (PC1)
        Direction where data varies the most.
  2. 2.Second Principal Component (PC2)
        Orthogonal to PC1 · captures second-most variance.
Maths behind PCA:—
  • PCA finds directions that maximize variance.

Optimization goal:

maxw Var (Xw)

Subject to: ‖w‖ = 1

This leads to Eigenvectors of Covariance Matrix.

#Step-by-Step: How PCA Works
1.Standardize the data: Mean = 0, Variance = 1
Why? So features don't dominate due to scale.
2.Compute Covariance Matrix:
C = 1n XTX

where XT = transpose of matrix · This tells how features vary together.

3.Compute Eigenvalues & Eigenvectors:
Eigenvectors → directions (principal components)
Eigenvalues → importance (variance)
4.Sort by Eigenvalues:
Highest eigenvalues → most important component.
5.Select top k components:
Reduce dimensions (eg: 100 → 10)
6.Transform data:
Project original data onto new axes.
Z = X · V₁
Simple Python Example:—
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

X_scaled = StandardScaler().fit_transform(X)
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)
print(pca.explained_variance_ratio_)
