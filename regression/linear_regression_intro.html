–Linear Regression is a supervised learning algorithm used to predict a continuous value.
–It is a statistical method used to model the relationship between a dependent variable (Y) and one or more independent (X) variables, assuming the relationship is linear.
–Basic idea: fit a straight line (or hyperplane) that minimizes the difference between actual and predicted values.
Simple Linear Regression y = mx + b
y: predicted value
x: input feature
m: slope / coefficient (weight) ★★
b: intercept (bias) ★★
Also: slope is also known as coefficient.
Multiple Linear Regression y = β₀ + β₁x₁ + β₂x₂ + … + βₚxₚ
β₀: intercept
βⱼ: slope / coefficient for feature j
Geometrical Intuition

Residual Error :-

•The Residual Error (ε) is the difference between the actual observed value (y) and the predicted value (ŷ) from the model.
Note: If Sum of Residual Error is minimum, that is called the best fit line of Linear Regression model.

If Sum of Residual Error is zero → it causes Overfitting problem.
Formula ŷ = mx + c     Residual Error → ½ Σ (Δᵢ)²
★Slope (m): If 1 unit increase in x-axis leads to how many units increase in y-axis.
★Intercept (b/c): When x value is zero — where the best fit line is intercepting the y-axis.

Key Components

# OLS Role :-

Note: In Linear Regression model our aim is to find best fit line with minimal error. Linear Regression has 3 key components:
Component Concept Role
Cost Function The measure of error Measures how bad our model is
OLS (Ordinary Least Squares) The model method Defines what "best" means mathematically
Gradient Descent The optimization tool Finds the best parameters efficiently

1. OLS (Ordinary Least Squares) :-

–It says: "Choose parameters such that the sum of squared errors is minimized."
–OLS defines the objective.
–It directly finds the parameter values (coefficient β, θ) to minimize the cost function — specifically the Sum of Squared Error (SSE).
SSE Formula SSE = Σi=1m ( y(i) − hθ(x(i)) )²
Closed-Form Solution (Normal Equation) θ̂ = (XᵀX)⁻¹ Xᵀy
X: The matrix of all input features
Xᵀ: The transpose of matrix X
⁻¹: The inverse of the matrix
Note: OLS is nothing but MSE without division by N.
OLS can resolve analytically (without iteration) and gives a closed-form solution.
# NumPy implementation of Normal Equation theta = np.linalg.inv(X.T @ X) @ X.T @ y # where: X.T = matrix of transpose feature, y = target vector # linalg = linear algebra
Section ii

Cost Function :-

–How wrong is my model. A math used to calculate how "wrong" the model is at a given moment.
–We typically use Mean Squared Error (MSE):
MSE / Cost Function J(θ) J(θ) = 1/(2m) · Σi=1m ( hθ(x(i)) − y(i) )²   ↓ minimize
hθ(x): The predicted value (the line)
y: The actual value
m: The number of data points
Why 1/2m ?
Because of the chain rule: ∂x²/∂x = 2x → the 2 cancels the ½, giving us a cleaner derivative.
To get the slope → use differentiation.
–(yactual − ypredicted) value can be +ve or −ve based on coefficient. To prevent −ve we use square.
–Cost function also known as Squared Error Function (SEF).
–We need to change coefficient value to change or minimize the cost function.
Example Calculation — for θ₀=0, θ₁=0.5 with data (1,1)(2,2)(3,3):
hθ predictions
h(x₁) = 0+1×0.5 = 0.5
h(x₂) = 0+2×0.5 = 1.0
h(x₃) = 0+3×0.5 = 1.5
J(θ₁) at θ₁=0.5
= 1/6 [(0.5−1)²+(1−2)²+(1.5−3)²]
= 1/6 [3.5] = 0.58
J(θ₁) at θ₁=1 → Perfect Fit
= 1/6 [(1−1)²+(2−2)²+(3−3)²] = 0.0 ← global minima
Section iii

Gradient Descent :-

–Gradient Descent is an optimization algorithm that iteratively adjusts model parameters in the direction of negative gradient of the cost function to minimize error.
–It tweaks the weight (coefficient) of the model step-by-step to "walk down" the hill of the cost function until it reaches the global minima (minimum error).
–It takes a shortcut — jumps directly to the bottom instead of walking step-by-step.
Analogy:
• Cost Function = hill
• Gradient Descent = walking downhill to reach minimum
• Minimum point = best values of m and c
• Best m & c = best fit line
Repeat Convergence Theorem (RCT) — Update Rule θⱼ := θⱼ − α · ∂/∂θⱼ J(θ)    ← slope or coefficient
α: Learning Rate — a small positive no. (e.g. 0.01) that determines the size of step taken down the gradient.
∂/∂θⱼ: The partial derivative — calculates slope of cost function.
And in Gradient Descent, global minima tells that model training has to happen near the global minima value.
Convergence Theorem: If the cost function is convex and the learning rate α is small enough, then gradient descent is guaranteed to converge to the global minimum.
Math Deep Dive

# Partial Derivatives :-

Simple Derivative
f(x) = x³
∂/∂x (x³) = 3x²
Partial Derivative
f(x,y) = x³ + y²
∂f/∂x = 3x² + 0
∂f/∂y = 0 + 2y
–Learning Rate (α) decides how fast or slow movement will happen in +ve or −ve gradient.
•Do not take too large or small α rate — it will not or miss the global minima if it is large.

How It Works?

  1. Start with random weight (slopes).
  2. Calculate the gradient (the derivative) of the cost function. This tells you direction of steepest increase.
  3. Take small step in the opposite direction & keep changing θ₀, θ₁ to reduce J(θ₀, θ₁) until we reach the global minima.
  4. Repeat until the cost function stops changing.
Putting it Together

Gradient Descent with m & b :-

Cost function with coefficient m and intercept b:
Cost Function J = 1/(2m) · Σi=1m ( yⁱ − (mxi + b) )²
Partial derivatives respectively:
∂J/∂m (w.r.t coefficient) ∂/∂m = 2/m · Σi=1m −xⁱ (yⁱ − (mxi + b))
∂J/∂b (w.r.t intercept) ∂/∂b = 2/m · Σi=1m −(yⁱ − (mxi + b))
Then we apply the update rules:
Update Rules (apply simultaneously) m := m − α · ∂/∂m
b := b − α · ∂/∂b

where α = learning rate = 0.01
Note: We need to always normalize & standardize the data feature before training.
# Gradient Descent implementation def gradient_descent(X, y, lr=0.01, epochs=1000):
m, b = 0, 0
n =len(y)
for _ inrange(epochs):
y_pred = m * X + b dm =(-2/n) *sum(X * (y - y_pred)) db = (-2/n) * sum(y - y_pred) m -= lr * dm b -= lr * db return m, b
