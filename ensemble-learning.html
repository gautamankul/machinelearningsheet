Cheat Sheet — Ensemble Learning in ML

Combine multiple weak learners into one predictive model to reduce bias/variance and improve accuracy.

What is Ensemble Learning? (Interview-ready answer)

Ensemble learning combines multiple weak models (learners) to produce a stronger single model. Typical goals are to reduce variance (bagging), reduce bias (boosting), or improve predictive performance (stacking). Ensembles work because independent errors from weak learners can cancel out when combined.

1. Bagging

Train N models on different bootstrap samples in parallel. Voting/averaging reduces variance.

2. Boosting

Train models sequentially; later learners focus on examples earlier learners misclassified. Weighted voting reduces bias.

3. Stacking

Train heterogeneous base learners and a meta-learner that takes their predictions as input to produce the final prediction.

Block diagrams (simplified)

Bagging
Create N subsets → train N weak models → vote → final prediction.
Boosting
Train sequentially with adjusted weights on examples → weighted vote → final prediction.
Stacking
Train weak learners on subset → predictions fed to meta-learner → final prediction.

Quick Tips for Interviews

  • Describe the intuition first (variance vs bias).
  • Mention popular algorithms: Random Forest (bagging), AdaBoost/GradientBoosting/XGBoost/LightGBM/CatBoost (boosting), and stacking for heterogeneous ensembles.
  • Talk about tradeoffs: compute cost, interpretability, overfitting risk.
