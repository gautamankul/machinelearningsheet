Clustering

Unsupervised · Group Discovery
Find natural groupings in data without predefined labels
Partition-Based
K-Means
Partitions data into K clusters by iteratively assigning points to the nearest centroid and recomputing centroids. Requires K to be specified upfront. Sensitive to initialization and outliers.
Formula
Objective: min Σₖ Σ_{x∈Cₖ} ‖x − μₖ‖² Assignment: Cₖ = {x : ‖x−μₖ‖ ≤ ‖x−μⱼ‖ ∀j} Update: μₖ = (1/|Cₖ|) · Σ_{x∈Cₖ} x
Hierarchical
Hierarchical Clustering
Builds a tree of clusters (dendrogram) by successively merging (agglomerative) or splitting (divisive) clusters. No need to specify K — cut the dendrogram at any level.
Formula
Linkage distances: Single: min d(a,b), a∈A, b∈B Complete: max d(a,b), a∈A, b∈B Average: (1/|A||B|) Σ d(a,b) Ward: min ΔWCSS after merge
Density-Based
DBSCAN
Identifies clusters as dense regions separated by low-density areas. Automatically detects noise/outliers. Finds arbitrary-shaped clusters. Requires ε (radius) and MinPts parameters.
Formula
Core point: |N_ε(x)| ≥ MinPts N_ε(x) = {y : d(x,y) ≤ ε} Border: in N_ε of core but < MinPts Noise: neither core nor border point
Density-Based
Mean Shift
Iteratively shifts each data point toward the local density maximum (mode) using a kernel function. Automatically determines the number of clusters. No K needed.
Formula
m(x) = Σ K(x−xᵢ)·xᵢ / Σ K(x−xᵢ) Shift: x ← m(x) (repeat until conv.) K(x) = Gaussian kernel: exp(−‖x‖²/2h²) h = bandwidth hyperparameter

"Kind Heroes Don't Misplace Samples"

Dimensionality Reduction

Feature Selection · Feature Extraction
Compress data into fewer dimensions while preserving key structure
Feature Extraction · Linear
PCA
Projects data onto orthogonal axes (principal components) that capture maximum variance. Unsupervised — ignores class labels. Used for compression, noise removal, and visualization.
Formula
Covariance: Σ = (1/n) XᵀX Eigen decomp: Σ·v = λ·v Project: Z = X · Vₖ Vₖ = top-k eigenvectors Var explained: λₖ / Σλ
Feature Extraction · Non-Linear
t-SNE
Maps high-dimensional data to 2D/3D by preserving local neighborhood structure. Excellent for visualization. Non-deterministic and computationally expensive for large datasets.
Formula
High-dim affinity: pⱼ|ᵢ = exp(−‖xᵢ−xⱼ‖²/2σᵢ²) / Σ Low-dim affinity (Student-t): qᵢⱼ = (1+‖yᵢ−yⱼ‖²)⁻¹ / Σ Loss: KL(P‖Q) = Σ pᵢⱼ·log(pᵢⱼ/qᵢⱼ)
Feature Extraction · Supervised
LDA — Linear Discriminant Analysis
Finds axes that maximize between-class separation while minimizing within-class scatter. Unlike PCA, LDA uses class labels — semi-supervised dimensionality reduction.
Formula
Maximize: J(w) = wᵀSᵦw / wᵀSᵥw Sᵦ = between-class scatter matrix Sᵥ = within-class scatter matrix Project: z = Wᵀx (W = top eigenvecs)

Association Rules

Pattern Mining · Market Basket
Discover frequent item co-occurrences and "if-then" purchase patterns
Frequent Itemset Mining
Apriori Algorithm
Generates frequent itemsets using the anti-monotone property: all subsets of a frequent itemset are frequent. Builds itemsets bottom-up, pruning infrequent candidates at each level.
Formula
Support: sup(A) = |T(A)| / |T| Confidence: conf(A→B) = sup(A∪B) / sup(A) Lift: lift(A→B) = conf(A→B) / sup(B) Apriori property: if sup(X) < min_sup → prune all supersets
Key metrics: Support, Confidence, Lift Lift > 1 = positive association
Frequent Itemset Mining
FP-Growth
Mines frequent itemsets without candidate generation using a compact FP-Tree structure. Scans the database only twice — significantly faster than Apriori on dense datasets.
Formula
1. Build FP-Tree from transactions (header table + compressed trie) 2. Mine conditional pattern bases for each frequent item 3. Build conditional FP-Trees and extract frequent itemsets No candidate generation needed
Faster than Apriori Same support/conf/lift metrics
Metric
Support
How frequently does an itemset appear across all transactions? Filters out rare, statistically insignificant patterns.
Formula
sup(A) = |transactions containing A| / |total transactions| Range: [0, 1] Higher = more frequent
Metric
Confidence
Given that A is purchased, how often is B also purchased? Measures the strength of the conditional relationship.
Formula
conf(A→B) = sup(A ∪ B) / sup(A) Range: [0, 1] Higher = stronger rule
Metric
Lift
How much more likely is B purchased when A is purchased, compared to random chance? Corrects confidence for the base popularity of B.
Formula
lift(A→B) = conf(A→B) / sup(B) lift > 1 → positive association lift = 1 → independent lift < 1 → negative association

Clustering Evaluation Metrics

Internal · No Ground Truth Needed
Measure cluster quality without labeled data
Cohesion + Separation
Silhouette Score
Measures how similar a point is to its own cluster vs. neighbouring clusters. Averaged over all samples. Higher values indicate well-separated, compact clusters.
Formula
s(i) = (b(i) − a(i)) / max(a(i), b(i)) a(i) = mean dist to same-cluster points b(i) = mean dist to nearest other cluster Score = mean s(i) over all points
Range: [−1, 1] ↑ Closer to 1 = better
Compactness / Separation Ratio
Davies-Bouldin Index
Average ratio of within-cluster scatter to between-cluster separation for each cluster pair. Lower is better — tighter, well-separated clusters score closer to zero.
Formula
DB = (1/k) · Σᵢ max_{j≠i} Rᵢⱼ Rᵢⱼ = (σᵢ + σⱼ) / d(μᵢ, μⱼ) σᵢ = avg dist of points in Cᵢ to μᵢ d(μᵢ,μⱼ) = dist between centroids
Range: [0, ∞) ↓ Lower is better
Inertia · Compactness
WCSS — Within-Cluster Sum of Squares
Sum of squared distances from each point to its cluster centroid. The objective minimized by K-Means. Used in the Elbow Method to select the optimal K.
Formula
WCSS = Σₖ Σ_{x∈Cₖ} ‖x − μₖ‖² Elbow method: plot WCSS vs K, pick K where decrease flattens Also called: Inertia
Range: [0, ∞) ↓ Lower is better
