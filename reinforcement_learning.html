Model-Free Methods

No Environment Model · Learn from Experience
Learn optimal behaviour directly from environment interactions
Value-Based · Off-Policy
Q-Learning
Learns the optimal action-value function Q*(s,a) by bootstrapping from the max future Q-value. Off-policy — learns optimal policy regardless of which actions are taken during exploration.
Formula
Q(s,a) ← Q(s,a) + α·[target − Q(s,a)] target = r + γ · max_{a'} Q(s',a') α = learning rate γ = discount factor Greedy policy: π(s) = argmax_a Q(s,a)
Value-Based · On-Policy
SARSA
On-policy TD control. Updates Q using the action actually taken by the current policy (not the greedy max). More conservative than Q-Learning — safer in risky environments.
Formula
Q(s,a) ← Q(s,a) + α·[target − Q(s,a)] target = r + γ · Q(s', a') a' sampled from current policy π (not greedy max — key difference from Q-Learning)
Value-Based · Deep RL
DQN — Deep Q-Network
Extends Q-Learning with a neural network approximator for Q(s,a). Uses experience replay and a separate target network to stabilize training. Foundation of modern deep RL.
Formula
Loss = E[(r + γ·max_{a'} Q̂(s',a';θ⁻) − Q(s,a;θ))²] θ⁻ = target network params (frozen) θ = online network params Replay buffer: sample random minibatch
Policy-Based
Policy Gradient (REINFORCE)
Directly optimizes the policy π_θ by ascending the gradient of expected cumulative reward. No value function needed. High variance — usually paired with a baseline to reduce variance.
Formula
Objective: J(θ) = E[Σ rₜ] Gradient: ∇J(θ) = E[Σ ∇log π_θ(aₜ|sₜ) · Gₜ] Gₜ = Σ_{k=t} γ^(k−t) · rₖ (return) Update: θ ← θ + α · ∇J(θ)
Actor-Critic
A2C / A3C
Combines a policy network (Actor) and a value network (Critic). The Critic reduces variance by replacing raw returns with advantage estimates. A3C runs parallel workers asynchronously.
Formula
Actor update: ∇J(θ) = E[∇log π_θ(a|s) · A(s,a)] Advantage: A(s,a) = Q(s,a) − V(s) ≈ r + γV(s') − V(s) (TD error) Critic loss: (r + γV(s') − V(s))²
Policy-Based · Clip
PPO — Proximal Policy Optimization
Improves upon TRPO by clipping the policy update ratio to prevent destructively large updates. State-of-the-art for continuous control, robotics, and game playing environments.
Formula
L_CLIP(θ) = E[min(rₜ·Â, clip(rₜ, 1−ε, 1+ε)·Â)] rₜ = π_θ(a|s) / π_θ_old(a|s) Â = advantage estimate ε = clip range (typically 0.1–0.2)

Model-Based Methods

Environment Model · Planning
Use or learn a model of the environment to plan ahead
Planning + Learning
Dyna-Q
Integrates model-free Q-Learning with model-based planning. After each real experience, Dyna-Q performs k simulated Q-Learning updates using its learned environment model.
Formula
Real experience: Q-Learning update Model: (s,a) → (s', r) learned Planning loop (k times): 1. Sample (s,a) from memory 2. Simulate: (s',r) = Model(s,a) 3. Q(s,a) ← Q(s,a) + α·[r+γ·max Q(s',·) − Q(s,a)]
Model Learning · Deep
World Models / MBPO
Learns a neural network model of the environment dynamics. Uses the model to generate synthetic rollouts for policy training, achieving high sample efficiency.
Formula
Learn: f_θ(sₜ, aₜ) ≈ P(sₜ₊₁ | sₜ, aₜ) Plan / Imagine: sₜ₊₁ ~ f_θ(sₜ, aₜ) Policy trained on model rollouts: π* = argmax_π E_model[Σ γᵗ rₜ]

Core RL Concepts

Foundations · Terminology
Fundamental building blocks underlying all RL algorithms
Value Functions
Bellman Equation
Recursive decomposition of value functions. States that the value of a state equals the immediate reward plus the discounted value of the next state. Foundation of all TD methods.
Formula
State value: V(s) = E[r + γ·V(s') | s] Action value: Q(s,a) = E[r + γ·max_{a'} Q(s',a') | s,a] Optimal: V*(s) = max_a Q*(s,a)
Policy Strategy
Exploration vs Exploitation
Core RL tradeoff — exploit known high-reward actions vs. explore unknown actions that might be better. ε-greedy balances both: take random action with probability ε, greedy otherwise.
Formula
ε-Greedy policy: π(s) = { random action w.p. ε argmax_a Q(s,a) w.p. 1−ε } ε decay: εₜ = ε₀ · decay^t UCB: a = argmax_a [Q(a) + c√(ln t / Nₐ)]
Problem Formulation
Markov Decision Process (MDP)
The mathematical framework for RL problems. Defined by states, actions, transition probabilities, reward function, and discount factor. Markov property: future depends only on current state.
Formula
MDP = (S, A, P, R, γ) P(s'|s,a) = transition probability R(s,a) = expected reward γ ∈ [0,1] = discount factor Return: Gₜ = Σ_{k=0}^∞ γᵏ · rₜ₊ₖ

Evaluation Metrics

Performance · Convergence
Measure how well an RL agent learns and performs
Primary Signal
Cumulative Reward
Total reward accumulated by the agent across an episode or training run. The primary optimization objective. Often plotted against training steps to show learning progress.
Formula
G = Σ_{t=0}^{T} rₜ (undiscounted) Discounted: G = Σ_{t=0}^{T} γᵗ · rₜ Episode return = G of one full episode
Task-dependent range ↑ Higher is better
Convergence Indicator
Average Episode Reward
Mean reward per episode over a rolling window (e.g. last 100 episodes). Smooths noise in individual episode returns. Standard benchmark metric in RL literature.
Formula
Avg Reward = (1/N) · Σᵢ Gᵢ N = number of evaluation episodes Typically computed over last 100 eps Common threshold: Avg ≥ target → solved
Task-dependent ↑ Stable high = converged
Efficiency
Sample Efficiency
How many environment interactions are needed to reach a target performance level. Model-based methods are typically more sample-efficient than model-free methods.
Formula
Steps to threshold: N* = min{t : Avg_reward(t) ≥ target} Area under learning curve: AUC = Σₜ Avg_reward(t) Lower N* = more sample-efficient
Steps to target ↓ Fewer steps = better
Convergence Quality
Policy Stability / Variance
Variance of episode returns across evaluation rollouts. High variance suggests the policy is unstable or hasn't fully converged. Low variance with high mean is ideal.
Formula
Std Dev = √[ (1/N) Σ (Gᵢ − Ḡ)² ] Coefficient of variation: CV = Std Dev / |Ḡ| Low CV + High Ḡ = stable, good policy
CV range: [0, ∞) ↓ Lower variance = better
Training Diagnostic
TD Error / Q-Value Convergence
Temporal difference error measures the gap between estimated and target Q-values. Decreasing TD error over training indicates the value function is converging toward the true values.
Formula
TD error δ = r + γ·Q(s',a') − Q(s,a) Loss (MSE): L = E[δ²] Convergence: |Q_new − Q_old| → 0 Monitor: plot mean |δ| vs training steps
Range: [0, ∞) ↓ Decreasing = converging
